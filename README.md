# Alembic — YZV 322E Tool Presentation

Burak Bayındır · 150200340 · ITU AI & Data Engineering · Spring 2026

Tool: Alembic (PostgreSQL / pgAdmin track).

This repo brings up PostgreSQL and a Python container with Alembic, applies three hand-written migrations, then autogenerates a fourth one from a model edit and rolls it back.

## 1. What is this tool?

Alembic is a Python migration framework for SQLAlchemy. Every schema change is a small Python file under `alembic/versions/`. You apply changes with `alembic upgrade head` and reverse them with `alembic downgrade`. License: MIT. First released 2010.

## 2. Prerequisites

- Docker Engine 24+ and Docker Compose v2
- ~500 MB free disk
- Host port 5433 free (mapped to Postgres 5432 inside the network)

No host Python required — everything runs in containers.

## 3. Installation

```bash
git clone https://github.com/Budi4/alembic-presentation.git
cd alembic-presentation
cp .env.example .env
docker compose up -d --build
```

Check both services are running:

```bash
docker compose ps
```

`alembic_demo_db` should report `healthy`, `alembic_demo_app` should report `running`.

## 4. Running the example

```bash
docker compose exec app bash demo.sh
```

This script runs eleven labelled steps. Briefly:

1. `alembic current` — empty on a fresh DB
2. `alembic history` — list pending revisions
3. `alembic upgrade head` — apply 0001 → 0002 → 0003
4. `alembic current` — confirm `0003 (head)`
5. `psql \dt` — list tables (`users`, `posts`, `alembic_version`)
6. Stage a model edit (copy `app/models_with_tag.py` over `app/models.py`)
7. `alembic revision --autogenerate -m "add tags table"`
8. Print the generated migration
9. `alembic upgrade head`
10. `alembic downgrade -1`
11. `alembic history --verbose`

Individual commands also work, e.g.:

```bash
docker compose exec app alembic current
docker compose exec app alembic history
docker compose exec db psql -U alembic_demo -d alembic_demo -c "\dt"
```

To reset between runs:

```bash
docker compose down -v
```

The autogenerated file uses a fresh hash on every run and is excluded from version control via `.gitignore`.
